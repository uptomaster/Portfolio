프로젝트 핵심 요약

WHAT

밥세권 서비스 전체 인프라(Linux · DBMS · PHP Web · Backup) 보안 재구축

WHY

침투 시나리오 기반 관리자 페이지 장악 → 서비스 중단 위험 확인

HOW

보안 점검(U-01~72) + 취약점 분석 + 백업/DB 이중화 + 웹 보안 강화

RESULT

보안수준 향상 · 안정화된 서비스 운영 · 재해복구 시간(RTO) 대폭 감소

본사·지사 서버 역할

HQ_DB_SEC

본사 MariaDB Master 서버 · DB 보안 정책 및 로그 감사 적용

HQ_SFTP_SEC

SSH Key 기반 백업 수신 · 부서별 격리 디렉토리 구성

HQ_LOG_SEC

로그 수집 및 분석 환경 구축 (syslog / logrotate / 접근 추적)

HQ_MAIL_SEC

Postfix 기반 사내 메일 시스템 · TLS 강화 및 인증 로그 분석

BR_BACKUP_SEC

지사 백업 서버 · HQ/BR 양방향 rsync 백업 허브

BR_PHP_SEC

밥세권 웹서비스 PHP 서버 · 관리자 페이지 보안 설정 강화
